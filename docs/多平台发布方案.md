# DoomLike 多平台发布方案（可执行单元版）

## Context

DoomLike 仅能在 Windows (D3D12/Forward+) 编辑器内运行，需发布到 **PC (Win/Linux/macOS)**、**Android**、**Web** 三平台。核心挑战：Web 强制 Compatibility (GLES3) 渲染器，Forward+ 不兼容 Web；Android 需完整触摸输入系统。

---

## 第一阶段：基础设施（可执行单元 × 6）

### 单元 1.1 — `project.godot` 添加窗口拉伸 + 版本号 + 特性标签修正

**文件**：`project.godot`

**操作**：7 个独立 Edit

| # | old_string | new_string | 说明 |
|---|-----------|-----------|------|
| 1 | `config/features=PackedStringArray("4.6", "Forward Plus")` | `config/features=PackedStringArray("4.6")` | 移除硬编码 Forward+，改运行时检测 |
| 2 | `config/icon="res://icon.svg"` | `config/icon="res://icon.svg"\nconfig/version="0.1.0"` | 添加版本号 |
| 3 | `window/size/viewport_height=720` | `window/size/viewport_height=720\nwindow/size/mode=2\nwindow/stretch/mode="canvas_items"\nwindow/stretch/aspect="expand"` | 窗口最大化 + UI 等比缩放 |
| 4 | `rendering_device/driver.windows="d3d12"` | `rendering_device/driver.windows="d3d12"\nrendering_device/driver.linuxbsd="vulkan"\nrendering_device/driver.macos="metal"\nrendering_device/driver.android="vulkan"\nrendering_device/fallback_to_gles.android=true\nrendering_device/driver.web="gles3"` | 多平台渲染驱动 |
| 5 | `3d/physics_engine="Jolt Physics"` | `3d/physics_engine="Jolt Physics"\n3d/physics_engine.web="GodotPhysics"` | Web 端回退 GodotPhysics（Jolt WASM 不稳定） |

### 单元 1.2 — 创建 `scripts/platform/platform_detector.gd`

**文件**：`scripts/platform/platform_detector.gd`

**内容**：~50 行，提供静态方法：
- `get_current() -> int` — 返回 Platform 枚举（PC_WINDOWS/PC_LINUX/PC_MACOS/ANDROID/WEB/UNKNOWN）
- `is_pc() -> bool`
- `is_mobile() -> bool`
- `is_web() -> bool`
- `is_touch_primary() -> bool` — Android=true，其他=false
- `get_renderer() -> String` — 调用 `RenderingServer.get_current_rendering_method()`

### 单元 1.3 — 创建 `scripts/platform/ui_scaler.gd`

**文件**：`scripts/platform/ui_scaler.gd`

**内容**：~30 行，挂载到 UI CanvasLayer 根节点，基于 DESIGN_WIDTH=1280 / DESIGN_HEIGHT=720 等比缩放，保持宽高比（极宽屏留黑边）。

### 单元 1.4 — 创建 `scripts/core/version.gd`

**文件**：`scripts/core/version.gd`

**内容**：~10 行，从 ProjectSettings 读取 `application/config/version` 并暴露 `get_version_string()`

### 单元 1.5 — 修改 `scripts/main.gd` 添加平台初始化

**文件**：`scripts/main.gd`

**操作**：在 `_ready()` 末尾追加平台初始化代码块：
- 实例化 `PlatformDetector`（如需要）
- 实例化 `UIScaler` 挂载到 UI CanvasLayer
- 条件检查：`if PlatformDetector.is_touch_primary():` 启用触摸 HUD（后续阶段连线）

### 单元 1.6 — MCP 验证

使用 `mcp__godot-mcp__audit_project_health` 检查项目完整性，确保上述修改无破坏。

## 第二阶段：PC 打包（可执行单元 × 3）

### 单元 2.1 — 创建 `export_presets.cfg`（PC 三个预设）

**文件**：`export_presets.cfg`

**内容**：包含 Windows Desktop、Linux/X11、macOS 三个导出预设。
- Windows: `export_path="build/windows/DoomLike.exe"`, `binary_format/embed_pck=false`
- Linux: `export_path="build/linux/DoomLike.x86_64"`
- macOS: `export_path="build/macos/DoomLike.zip"`

### 单元 2.2 — 添加 `[input_devices]` 配置到 `project.godot`

**文件**：`project.godot`

**操作**：添加：
```ini
[input_devices]
pointing/emulate_touch_from_mouse=false
pointing/emulate_mouse_from_touch=false
```

### 单元 2.3 — MCP 验证导出配置

使用 `mcp__godot-mcp__validate_export_preset` 验证 Windows Desktop 预设可用。

## 第三阶段：Web 适配（可执行单元 × 7）

### 单元 3.1 — 渲染器兼容性检查脚本

使用 `mcp__godot-mcp__execute_editor_script` 运行脚本，检查项目中所有 `.tres` 材质文件是否使用 Compatibility 兼容的特性（检查 shader 类型、render_mode 等），列出不兼容项。

### 单元 3.2 — Web 预设添加到 `export_presets.cfg`

在 `export_presets.cfg` 追加 preset，平台为 "Web"，路径 `build/web/index.html`，使用自定义 HTML shell `res://web/web_shell.html`。

### 单元 3.3 — 创建 `web/web_shell.html`

**文件**：`web/web_shell.html`

**内容**：完整的 Godot 4.x HTML 导出模板，包含：
- 加载进度条（`<div id="progress-bar">`）
- 全屏按钮（通过 `canvas.requestFullscreen()`）
- 响应式容器（Canvas 自适应窗口）
- 自定义背景和加载提示
- `$GODOT_BASIC` 占位符（Godot 导出时替换）

### 单元 3.4 — 创建 `web/manifest.json`

**文件**：`web/manifest.json`

**内容**：PWA 清单，定义 name="DoomLike"、display=fullscreen、orientation=landscape

### 单元 3.5 — 创建 `web/service_worker.js`

**文件**：`web/service_worker.js`

**内容**：基础 PWA Service Worker，缓存 `.wasm`、`.pck`、`.js`、`.html` 等核心文件

### 单元 3.6 — 创建 Web 平台条件兼容脚本

**文件**：`scripts/platform/web_export_compat.gd`

**内容**：~30 行，如果 `PlatformDetector.is_web()`，在 `_ready()` 中将所有 StandardMaterial3D 材质切换为 `SHADING_MODE_UNSHADED`（兼容模式无光照效果更干净）

### 单元 3.7 — MCP 验证 Web 场景

使用 `mcp__godot-mcp__run_project` 运行项目，使用 `mcp__godot-mcp__get_runtime_screenshot` 截图，人工验证视觉效果。

## 第四阶段：Android 适配（可执行单元 × 8）

### 单元 4.1 — Android 预设添加到 `export_presets.cfg`

在 `export_presets.cfg` 追加 preset，平台 "Android"，路径 `build/android/DoomLike.apk`，配置：
- `graphics/opengl_debug=false`
- `screen/immersive_mode=true`
- `screen/support_splash=true`
- `xr_features/enable_xr=false`

### 单元 4.2 — 创建 `scripts/platform/touch_input.gd` — 核心部分

**文件**：`scripts/platform/touch_input.gd`

**内容**：~250 行，完整的虚拟触屏 HUD。

**架构**：
```
TouchInput (Control 根节点)
├── MoveJoystick (自定义 Control，左半屏)  — 虚拟摇杆
├── LookArea (自定义 Control，右半屏)     — 视角滑动
├── ButtonPanel (VBoxContainer，右下角)
│   ├── BtnPrimaryFire (TextureButton)     — 主射击
│   ├── BtnJump (TextureButton)            — 跳跃
│   ├── BtnWhip (TextureButton)            — 铁鞭
│   ├── BtnDash (TextureButton)            — 冲刺
│   ├── BtnAction (TextureButton)          — 交互/处决
│   └── BtnReload (TextureButton)          — 装弹
└── WeaponBar (HBoxContainer，顶部)
    ├── BtnWeapon1~4 (TextureButton)       — 武器切换
```

**核心逻辑**：
- 所有按钮通过 `_make_action_event(action, pressed, strength)` → `Input.parse_input_event()` 注入虚拟输入
- 虚拟摇杆四方向映射到 `move_forward/back/left/right`，支持强度（strength=摇杆偏移量）
- 视角滑动映射到 `InputEventMouseMotion` 注入
- 代码生成按钮纹理（用 Godot 内置绘图 API 画简单图形，无需外部纹理文件）

### 单元 4.3 — 创建 `scripts/platform/touch_button_painter.gd`

**文件**：`scripts/platform/touch_button_painter.gd`

**内容**：~60 行，静态工具类，用代码生成虚拟按钮的 ImageTexture（圆形、方形按钮，带颜色和透明度），供 `touch_input.gd` 使用。避免外部纹理依赖。

### 单元 4.4 — 修改 `scripts/main.gd` 连线触摸 HUD

**文件**：`scripts/main.gd`

**操作**：在平台初始化代码块中，实例化 `TouchInput` 并添加为 UI CanvasLayer 子节点：
```gdscript
if PlatformDetector.is_touch_primary():
    var touch_input := preload("res://scripts/platform/touch_input.gd").new()
    %UI.add_child(touch_input)
```

### 单元 4.5 — Android 性能优化：敌人数量动态调整

**文件**：`scripts/enemy/spawn_manager.gd`

**操作**：在 `_ready()` 中添加 Android 端同时最大敌人数限制为 15（PC 保持原值）。

### 单元 4.6 — Android 性能优化：敌人距离剔除

**文件**：`scripts/enemy/enemy.gd`

**操作**：在 `_ready()` 中为 `MeshInstance3D` 子节点设置 `visibility_range_end = 15.0`（15米外隐藏非关键模型）。

### 单元 4.7 — 创建 `scripts/build/build_all.ps1`

**文件**：`scripts/build/build_all.ps1`

**内容**：PowerShell 脚本，接收 `-Platform` 参数（windows/linux/macos/android/web/all），调用 Godot 无头模式逐平台导出。Android 需额外传递 keystore 参数。

### 单元 4.8 — MCP 验证场景载入

使用 `mcp__godot-mcp__run_project` + `mcp__godot-mcp__get_runtime_scene_tree` 验证新脚本不会导致崩溃。

## 第五阶段：CI/CD 自动化（可执行单元 × 2）

### 单元 5.1 — 创建 `.github/workflows/build_all.yml`

**文件**：`.github/workflows/build_all.yml`

**内容**：~120 行，GitHub Actions 工作流。
- **trigger**: push to main + manual workflow_dispatch
- **Job 1** `build-windows`: `windows-latest`, 下载 Godot 4.6, 导出 Windows Desktop
- **Job 2** `build-linux`: `ubuntu-latest`, 安装 Vulkan 依赖, 导出 Linux/X11
- **Job 3** `build-macos`: `macos-latest`, 导出 macOS
- **Job 4** `build-android`: `ubuntu-latest`, 设置 Java 17 + Android SDK, 导出 APK (debug keystore)
- **Job 5** `build-web`: `ubuntu-latest`, 导出 Web, 上传 artifact, 自动部署 GitHub Pages
- 每个 job 上传 artifact（`actions/upload-artifact`）

### 单元 5.2 — 验证 CI 配置文件语法

用 `mcp__godot-mcp__execute_editor_script` 检查所有导出预设名称与 CI 中引用的名称一致。
