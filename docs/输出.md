# Blender → Godot 建模工作流（配合 Claude Code + DeepSeek）

> 目标：用 Blender 制作 3D 模型，导出 glTF 2.0，在 Godot 中用 Claude Code 完成脚本挂载、碰撞设置、材质调整。只学必需的最小操作集，不需要学全套 Blender。

---

## 1. 格式选择：只认 glTF 2.0

Godot 官方推荐的 3D 格式是 **glTF 2.0**（.gltf 文本 + .bin + 贴图，或 .glb 单文件打包）。它保留网格、材质槽、骨骼动画，不需要第三方插件。

| 对比   | glTF 2.0 | OBJ    | FBX               |
| ------ | -------- | ------ | ----------------- |
| 材质槽 | 支持     | 不支持 | 支持              |
| 骨骼动画 | 支持   | 不支持 | 支持（但 Godot 支持差） |
| Godot 官方推荐 | 是 | 否   | 否                |
| 导出难度 | 一键   | 一键   | 复杂              |

**导出操作**：Blender → File → Export → glTF 2.0，勾选：

- Format: glTF Binary (.glb) — 单文件，方便管理
- Apply Modifiers — 必须勾，塌陷修改器
- Include → Selected Only — 按需导出，避免导出场景垃圾
- 静态道具不选 Animation

## 2. Blender 最小操作集

### 2.1 六组必备操作

| 操作             | 快捷键            | 说明                                       |
| ---------------- | ----------------- | ------------------------------------------ |
| 物体模式 / 编辑模式 | Tab             | 最基本的状态切换                            |
| 移动 / 旋转 / 缩放 | G / R / S       | 按 X / Y / Z 锁定轴向                      |
| 挤出             | E                 | 从面/边拉出新几何体，建模核心操作             |
| 环切             | Ctrl+R            | 给面加细分线，滚动滚轮调段数                  |
| 合并顶点         | M                 | 选择后按距离合并，清理重叠顶点                |
| Apply Scale      | Ctrl+A → Scale    | 导出前必做，否则 Godot 缩放错乱              |

### 2.2 常用辅助操作

| 操作             | 快捷键                       | 说明                                       |
| ---------------- | ---------------------------- | ------------------------------------------ |
| 环选边           | Alt+左键 点击边               | 选一圈边                                    |
| 内插面           | I                            | 在面内生成缩小的面                           |
| 倒角             | Ctrl+B                       | 选边后倒角，滚轮调段数                       |
| 分离选中         | P                            | 把选中的面/体分离为独立物体                   |
| 原点归几何中心   | F3 → Origin to Geometry      | 原点偏了导出后 pivot 会错                    |
| 翻转法线         | Alt+N → Flip                 | 面变黑了就是法线反了                          |
| Shade Auto Smooth | 右键 → Shade Auto Smooth    | 低模平滑显示                                 |
| 指定材质槽       | 材质面板 → + → Assign         | 不同面可设不同材质槽（Godot 按槽读取）         |

### 2.3 建模常识（防坑）

- **四边面**：尽量保持四边面，三角面不要太多，五边以上面 Godot 会帮你三角化但结果不可控
- **Apply Scale 必做**：一旦在物体模式缩放过，Ctrl+A → Scale 必须在导出前做，否则 Godot 里物体显示和碰撞尺寸都会错
- **原点位置**：原点在物体底部中心 → 适合放在地面上；原点在几何中心 → 适合浮空/手持物体
- **命名规范**：Blender 里物体名 = Godot 里 mesh 名，用英文命名（如 rifle_body、barrel_left）
- **不要 0 厚度**：平面从侧面看是透明的（背面剔除），需要厚度 → E 挤出

## 3. Godot 导入设置

把 .glb 拖入 Godot 文件系统后，选中文件，右侧 Import 标签页：

### 3.1 关键设置项

| 设置               | 推荐值                                       | 说明                                       |
| ------------------ | -------------------------------------------- | ------------------------------------------ |
| Root Type          | 静态/武器选 Node3D，敌人选 CharacterBody3D     | 决定生成场景的根节点类型                      |
| Generate Physics   | 简单形状勾选                                   | 自动给 mesh 加碰撞，复杂模型不勾手动加         |
| Collider Type      | 静态用 Trimesh，动态用 Convex                  | Trimesh 适合地形掩体，Convex 适合可移动物体    |
| Import Animation   | 有骨骼动画才勾                                 | 没动画的静态模型勾了会报 warning              |
| Embedded Images    | 勾选                                          | 材质贴图跟着 .glb 走                         |

### 3.2 导入后检查清单

- [ ] 模型尺寸正确（Blender 1m = Godot 1m，比例应该是 1:1）
- [ ] 材质槽数量正确（Blender 里几个材质，Godot 里就有几个 surface）
- [ ] 碰撞尺寸和模型匹配
- [ ] 原点 / pivot 位置合理
- [ ] 法线方向没有黑面

## 4. 材质工作流

### 4.1 Blender → Godot 材质映射

glTF 只传递最基础的 PBR 通道，Blender 节点图不会被导出：

| Blender（Principled BSDF） | Godot（StandardMaterial3D） | 说明     |
| -------------------------- | --------------------------- | -------- |
| Base Color                 | Albedo                      | 基础颜色 |
| Metallic                   | Metallic                    | 金属度   |
| Roughness                  | Roughness                   | 粗糙度   |
| Normal（Bump）              | Normal                      | 法线贴图 |
| Emission                   | Emission                    | 自发光   |
| Alpha                      | Transmittance               | 透明度   |

### 4.2 最佳实践

- **Blender 侧**：只设 Base Color 和 Roughness，用一眼能看出的颜色区分不同材质槽
- **Godot 侧**：用 Claude Code 帮你精细调整 StandardMaterial3D 参数
- **复杂 shader**：在 Godot 里写 .gdshader，不走 Blender 导出

### 4.3 让 Claude Code 帮你做

> "把这个模型的 Albedo 改成暗红色，Metallic 0.3，Roughness 0.8"
>
> "给 surface 0 加 emission 做发光效果"
>
> "把材质换成 res://assets/materials/enemy_armor.tres"

## 5. 实战工作流（Claude Code 协作）

```
Blender 建模 → 导出 .glb → 拖入 res://assets/models/
  ↓
Claude Code（通过 Godot MCP）帮你:
  - 调整 import 设置
  - 创建 / 附加脚本到模型节点
  - 设置 CollisionShape3D 碰撞体
  - 调整材质参数、替换材质资源
  - 把模型挂到场景里（替换现有 CSG 占位）
  - 运行测试验证
```

## 6. 本项目可立即着手的工作（按难度排序）

### 6.1 武器模型替换（推荐第一个做）

替换 WeaponHolder 下现有 CSG 武器：

| 武器   | CSG 现状                 | Blender 替代                               | 难度 |
| ------ | ------------------------ | ------------------------------------------ | ---- |
| 步枪   | CSG 长枪造型             | 步枪模型，挂在 WeaponHolder/Rifle            | 低   |
| 霰弹枪 | 木质枪身 + 双并排枪管     | 双管猎枪模型，挂在 WeaponHolder/Shotgun      | 低   |
| 手枪   | CSG 方块                 | 手枪模型，挂在 WeaponHolder/Pistol          | 低   |
| 拳头   | 无模型                   | 拳套 / 空手标记                             | 低   |

**建模要点：**

- 从摄像机位置出发，枪口在屏幕下方偏右
- 枪口需要标记一个空物体（empty）作为 muzzle_point，供弹道线/子弹起点
- 武器不需要碰撞体（玩家手持）
- 参考尺寸：步枪长约 1.5m ~ 2m 在 Blender 里

### 6.2 敌人模型替换

替换 scenes/enemies/ 下各敌人 CSG 身体部件：

| 敌人               | 现用           | Blender 替代     | 含骨骼动画？    |
| ------------------ | -------------- | ---------------- | --------------- |
| imp（小恶魔）      | CSG 立方体     | 小恶魔模型       | 可选（先静态）  |
| demon_soldier      | CSG 方块       | 恶魔士兵         | 可选            |
| standard_enemy     | CSG 身体       | 标准敌人         | 可选            |
| 8 类 Phase 6 敌人  | CSG 颜色区分   | 对应模型         | 可选            |

**建模要点：**

- 敌人锚点在脚下（底部中心），方便放地面上
- 需要包含 Damageable + CollisionShape3D (Capsule) + 脚本 — 这些 Claude Code 帮你加
- 头顶预留空间放眩晕条（约 0.3m ~ 0.5m 上方）
- 尺寸参考：普通敌人高 1.8m ~ 2m，精英 2.2m ~ 2.5m

### 6.3 道具 / 掉落物模型

| 道具     | Blender 替代       | 说明                       |
| -------- | ------------------ | -------------------------- |
| 血包     | 红色医疗包 / 血瓶  | Pickup 基类，悬浮旋转       |
| 弹药箱   | 弹药盒             | ammo_pickup.gd             |
| 护甲碎片 | 护盾 / 胸甲碎片    | armor_pickup.gd            |

**建模要点：**

- 道具锚点在底部中心
- 小尺寸：约 0.3m ~ 0.5m 高
- 需要 Area3D + CollisionShape3D — Claude Code 帮你加
- 不需要自己在 Godot 写旋转/悬浮脚本（已有 Pickup 基类）

### 6.4 场景道具

| 道具   | 现用               | Blender 替代               |
| ------ | ------------------ | -------------------------- |
| 枯树   | CSG 棕色方块搭成   | 枯树模型，挂在 PropsRoot    |
| 岩柱   | CSG 深灰方块堆叠   | 粗糙岩石柱模型              |
| 边界柱 | CSG 细高方块       | 图腾柱 / 木桩（可选）       |

**建模要点：**

- 锚点在底部中心
- 需要有碰撞（StaticBody3D 或 Trimesh 碰撞），能挡玩家和子弹
- 枯树参考尺寸：高 3m ~ 5m，树干直径 0.3m ~ 0.6m
- 岩柱参考尺寸：高 2m ~ 4m，直径 0.5m ~ 1m

## 7. Blender 键盘记忆口诀

```
Tab 进出编辑        G 抓起来移动
R 旋转 S 缩放       E 从面挤出来
Ctrl+R 切环         M 合并顶点清垃圾
Ctrl+A Apply Scale  不做出门就后悔
```

---

## 8. 下一个推荐起点

```
优先级：武器模型 → 掉落物 → 枯树/岩柱 → 敌人模型 → 边界柱

理由：武器始终在屏幕前方，最容易被注意到；
     掉落物是简单几何体，适合练手；
     敌人模型最后做，因为和脚本/AI/动画耦合最深
```
