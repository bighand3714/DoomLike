已推送。以下是 5 项仍需手动完成的步骤说明：

1. Android 图标
在 res:// 下放两个 PNG 文件，然后在 export_presets.cfg 的 [preset.3] 段中指定：


# 需要准备的图标文件（放在项目根目录或 res:// 下）:
#   icon_192.png  — 192×192 像素
#   icon_512.png  — 512×512 像素

# 生成方式：用任意图像工具将 icon.svg 导出为 PNG
# 或在线工具: https://makeappicon.com
Godot 编辑器内配置路径：项目 → 导出 → Android → 选项 → 图标，分别设置主图标和自适应图标。

2. Android Release Keystore
在终端运行以下命令生成签名密钥：


keytool -genkey -v -keystore doomlike.keystore -storepass <你的密码> \
  -alias doomlike -keypass <你的密码> \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -dname "CN=YourName, OU=Dev, O=DoomLike, L=City, S=State, C=CN"
然后在 Godot 编辑器 导出 → Android → 选项 → Release 中填写 keystore 路径和密码。CI 自动构建用 debug keystore，发布正式版需手动导出。

3. Android SDK 路径
在 Godot 编辑器中：编辑器 → 编辑器设置 → 导出 → Android，确认：

Android SDK 路径 指向已安装的 SDK（如 C:\Android\Sdk）
JDK 路径 指向 JDK 17+
需预先安装 Android Studio 或命令行 SDK 工具
若尚未安装：


# 推荐安装 Android Studio，然后在 SDK Manager 中下载:
#   Android SDK Platform 34
#   Android SDK Build-Tools 34.0.0
#   NDK 25.2.9519653
4. GitHub Pages
打开你的仓库页面：https://github.com/bighand3714/DoomLike
点击 Settings → 左侧 Pages
Source 选 Deploy from a branch
Branch 选 gh-pages，目录 / (root)，保存
CI（build_all.yml）在 master 分支每次 push 时会自动将 Web 构建输出部署到 gh-pages 分支
访问地址为 https://bighand3714.github.io/DoomLike/
5. Compatibility 模式测试
在 Godot 编辑器中：

右上角渲染器下拉菜单 → 选择 Compatibility
运行主场景 res://scenes/main.tscn
重点检查：
CSG 敌人几何体是否正常渲染
材质颜色是否变暗（如过暗，web_export_compat.gd 会自动切换为 UNSHADED 模式）
铁鞭线条是否可见
天空/雾效是否正常
记录视觉差异，必要时调整材质 albedo_color / emission 补偿光照损失
